### Root Cause

The test13.in file intentionally does NOT use `global seed` keyword:
```python
seed = 19260817

def rand():
    seed += seed * 131072  # No "global seed" here!
    # ...
```

This is because:
1. Our interpreter doesn't support `global` keyword (parse error)
2. The test is designed to verify implicit global variable handling
3. Our implementation has a bug - global variables aren't properly modified in function scopes

### Why This Matters

The Pollard-Rho algorithm depends on a pseudo-random sequence generated by modifying `seed`. When `seed` isn't properly updated across function calls, the algorithm finds different factors, producing 162 lines instead of 48.

### What test13_fixed.in Shows

There's a `test13_fixed.in` that adds `global seed` on line 47. This file:
- Runs correctly in Python (48 lines)
- Produces parse error in our interpreter (we don't support `global`)
- Confirms that with proper global handling, the output should be 48 lines
